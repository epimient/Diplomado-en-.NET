# Ejemplo Guiado — Presentación Final

## Estructura sugerida de la presentación (5-7 min)

### 1. Introducción (1 min)
- Nombre del proyecto
- Integrantes del equipo
- Breve descripción

### 2. Tecnologías (1 min)
- .NET 8 LTS, C#, ASP.NET Core
- Entity Framework Core + SQLite
- Swagger
- Git + GitHub

### 3. Demostración en vivo (3 min)
- Mostrar el proyecto funcionando
- Probar endpoints con Swagger
- Crear, listar, actualizar, eliminar registros

### 4. Código destacado (1 min)
- Mostrar el modelo de datos (clases)
- Mostrar un controlador con sus endpoints
- Explicar la relación entre entidades

### 5. Lecciones aprendidas (1 min)
- ¿Qué fue lo más difícil?
- ¿Qué aprendieron nuevo?
- ¿Qué mejorarían?

## Checklist antes de presentar

- [ ] Código compila sin errores (`dotnet build`)
- [ ] API corre correctamente (`dotnet run`)
- [ ] Swagger funcional en `/swagger`
- [ ] README.md completo
- [ ] Capturas de pruebas guardadas
- [ ] Repositorio en GitHub actualizado
- [ ] Video de funcionamiento grabado
- [ ] Diapositivas listas
